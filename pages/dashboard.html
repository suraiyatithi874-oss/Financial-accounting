1. Accounting Dashboard

Purpose

The Accounting Dashboard is your central hub. It provides quick access to organizational financial records, enabling smooth transitions between accounts, vouchers, and reporting tools.

×

Dashboard Highlights

  • Centralized Access: Real-time overview of financial health.
  • Module Shortcuts: Direct links to COA, Vouchers, and Reconciliations.
  • System Status: Quick view of pending vs. authorized transactions.

1.2 Navigation Overview

The system utilizes a Sidebar Menu (Left Panel) for primary navigation. This allows users to switch between different functional areas without losing context.

Sidebar Module Explanation

Module Description
Chart of Accounts Define account structures (Assets, Liabilities, Income, Expenses).
Generate Vouchers Record daily transactions (Debit, Credit, Journal, Contra).
Bank Reconciliation Match system records with bank statements.
Income & Expense Voucher Record operational income and expense entries such as utilities, salaries, rent, service income, and adjustments.
Process Handle Period Endings (Day, Month, and Year Close).
Reports Generate Ledgers, Trial Balances, and Financial Statements.

1.3 First Login & Security

Accessing the System

  1. Open the system URL in a supported web browser.
  2. Enter your User ID and Password.
  3. Select your assigned Working Branch (e.g., Head Office or Outlet 01).
  4. Important: You must change your default password upon the first successful login.

Security Notice

Unauthorized access is strictly prohibited. Ensure your credentials are kept confidential and always "Log Out" when leaving your workstation.

1.4 Initial Configuration

Before recording transactions, ensure the following core configurations are verified by an Admin:

1. Company Profile

Verify Fiscal Year settings and Base Currency (e.g., BDT/USD).

2. Branch Mapping

Ensure all operational branches and warehouses are correctly defined.

3. User Roles

Confirm Maker/Checker permissions for voucher authorization.

4. Opening Balances

Input initial balances via Process → Opening Balance to start the period.
