6.2 Day Close

The Day Close process locks all financial activities for a specific date to ensure daily data integrity.

Navigation Path: Process → Day End

  • Select the relevant Branch and Date Range from the filters.
  • Click Search to display all day-end records for the selected period.
  • Review the grid to confirm SL, Branch, and Branch Date.
  • Click the Process button to finalize and lock transactions.

üóëÔ∏è Delete Option

Allows users to remove specific Day End records if adjustments are needed. Represented by the trash can icon in the Action column.

×

6.3 Month Close

Month-end closing is required to generate finalized monthly Trial Balances and P&L Statements.

Navigation Path: Process → Month End

  • Choose the appropriate Branch and Date Range from the dropdown filters.
  • Review records to confirm all entries for the month are finalized.
  • Click the Process button to close the month.

üóëÔ∏è Delete Option

Use the trash can icon in the Action column to remove Month End records if the month needs to be re-opened for adjustments.

×

6.4 Year Close

The Year Close is the final process that prepares the system for the next financial year.

Navigation Path: Process → Year End

  • Select the Branch and Date Range.
  • Confirm that all financial modules (vouchers, reconciliations, etc.) are fully closed.
  • Click the Process button to execute the final closure.

üóëÔ∏è Delete Option

Allows users to remove specific Year End records via the trash can icon in the Action column.

×

Critical Requirement

Closing operations are sequential. You cannot perform a Month Close until all days within that month have been successfully Day Closed.

6.5 Day Back Script

The Day Back Script allows users to shift a specific branch's operating date backward for administrative adjustments.

Navigation Path: Menu → Process → Day Back Script

  • Select Branch and Back Type: Choose the target outlet (e.g., 0001 - Outlet 01) and select Day Back to shift the date.
  • Choose the Date: Enter the "From" date to reset the branch (e.g., 16-07-2025). The system displays the current date for reference.
  • Apply the Change: Click the Back to Date button to confirm and execute the date reset.

üí° Reference Info

Always verify the current branch date displayed by the system before executing the reset to avoid data overlaps.

×

6.6 Day End Script

The Day End Script is used to finalize daily operations, locking transactions to ensure data integrity for the selected period.

Navigation Path: Menu → Process → Day End Script

  • Select Branch and End Type: Choose the outlet you want to close (e.g., 0002 - Outlet 02) and select Day End to initiate the process.
  • Set the Target Date: Enter the To Date you want to close up to (e.g., 17-07-2025). The system will display the current branch date for reference.
  • Confirm the Day End: Click the End to Date button. Once finalized, all transactions up to this date will be locked.

üîí Security Note

Performing a Day End prevents any further modifications to vouchers or sales for the closed period. Ensure all entries are audited before proceeding.
